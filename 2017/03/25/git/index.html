git常用的几个命令

发表于 2017-03-25 |

项目使用git作为版本控制工具，记录几条常用的命令，足够正常使用。

新建配置相关

在当前目录新建一个Git代码库

git init

新建一个目录，将其初始化为Git代码库

git init [project-name]

下载一个项目和它的整个代码历史

git clone [url]

设置提交代码时的用户信息

git config [–global] user.name “[name]”
git config [–global] user.email “[email address]”

拉取相关

将线上版本同步到本地但不会自动merge；

git fetch

将线上版本同步到本地并merge，若有未缓存文件会提示将被覆盖并停止同步；

git pull

查看该文件与线上版本的不同

git diff filepath

提交相关

查看当前目录中跟踪已修改和未跟踪的文件

git status

将该文件加入缓存

git add filepath

将该文件提交到本地，并提交相关描述

git commit filepath -m “describe”

将本地缓存文件提交至线上服务器

git push

回溯相关

恢复暂存区的指定文件到工作区

git checkout [file]

重置暂存区的指定文件，与上一次commit保持一致，但工作区不变

git reset [file]

重置暂存区与工作区，与上一次commit保持一致

git reset –hard
